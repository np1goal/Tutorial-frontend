Introduction

When a web page is loaded the browser creates a Document Object Model of the page. The object model lets Javascript to create dynamic HTML.

It can change HTML elements and attributes and CSS styles. It can add and remove HTML elements and attributes. It can react to HTML events in the page. It can also create new HTML events.

Methods

HTML DOM methods are actions that can be performed. HTML DOM properties are values that can be set or changed.

Document

Finding HTML elements

  • document.getElementById("id")
  • document.getElementByTagName("name")
  • document.getElementByClassName("name")

Changing HTML elements

  • element.innerHTML = change the inner content of HTML elements
  • element.style.property = change the style of an element
  • element.attribute = change attribute values of an HTML element --> Does not work in Typescript
  • element.setAttribute(attribute, value) = change the attribute of the value

The alignment of h5 is center, click on it, the alignment will change using element.setAttribute("align", "left")

Adding and Deleting HTML elements

  • document.createElement(element) = create HTML element
  • document.removeChild(element) = remove an HTML element
  • document.appendChild(elements) = add an HTML element
  • document.replaceChild(new, old) = replace an HTML element
  • document.write(text) = write into HTML output stream --> Will reload entire page with new content

Adding Event Handlers

  • document.getElementById("id").onclick = function { ...code... }

Elements

HTML elements can be found by id name, tag name, class name, CSS selectors and HTML object collections.

CSS Selectors

To find all HTML elements that matched a specified CSS selector (id, class names, types, attributes, values of attributes etc.) use querySelectorAll() method.

For example, to get call elements of p with class="intro" do:

var x = document.querySelectorAll("p.intro");

Events

There are events that can be added in HTML elements that can trigger some function.

Event Description
onclick When an element is clicked
onload When web page is loaded
  • Can be used to check user's browser type and browser version and accordingly load page
  • Can be used for restoring cookie information
onunload When web page is unloaded
  • Can be used for storing cookie information
onchange Mostly used with input tag
onmouseover When mouse is hovering the element
onmouseout When mouse is not hovering the element
onmousedown When mouse is pressed
onmouseup When mouse is released
onfocus When an element is focused on like clicking in textfield

Event Listener

The addEventListener() method attaches One or Many event handlers to a specific element.

Event Listeners can be added to any DOM object (like window) or HTML elements. It makes it easier to control how the event reacts to bubbling.

The removeEventListener() is used to remove an event listener.

You can add many event handlers of the same type to one element, i.e two "click" events. Many Event Listener can be attached to same element.

element.addEventListener(event, function, useCapture) --> event is like "click", notice it is not onclick; useCapture is optional

window.addEventListener("resize", function)

If <p> element is inside <div> element, then two options are possible - Bubbling or Capturing. In Bubbling the event listener of innermost element is executed first. In Capturing the event listener of outermost element is executed first.

The removeEventListener() method removes event handlers that have been attached with the addEventListener() method

NOTE: addEventListener() and removeEventListener() methods are not supported in IE 8 and earlier version. attachEvent() and detachEvent() can be used in its place.

Node Relationships

<html>
  <head>
    <title>DOM Tutorial</title>
  </head>
  <body>
    <h1>DOM Lesson one</h1>
    <p>Hello World!</p>
  </body>
</html>

Here, html is root node and has 2 children head (first child) and body (last child.)

Head has one child title and body has two children h1 and p (h1 and p are also siblings).

Title has one text node.

This nodes can be accessed as document.getElementById("demo").firstChild.nodeValue; or document.getElementById("demo").childNodes[1].nodeValue;

DOM Root Nodes

There are two special properties that allow access to the full document.

  • document.body - body of document
  • document.documentElement - full document

nodeName Property

nodeName property specifies the name of node.

  • nodeName is read-only
  • nodeName of element node is tag name (tag name is Uppercase)
  • nodeName of attribute node is attribute name
  • nodeName of text node is #text
  • nodeName of document node is #document

nodeValue Property

nodeValue property specifies the value of node.

  • nodeValue for element nodes is null
  • nodeValue of text nodes is the text itself
  • nodeValue of attribute nodes is attribute value

nodeType Property

nodeType property is read only and returns the type of node.

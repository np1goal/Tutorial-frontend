Scope

  • Global Scope
    • A variable declared outside all the functions are Global.
    • All scripts and functions on a web page can access it.

  • Local Scope
    • A variable declared inside the function is Local to the function.
    • They can only be accessed from within the function.
    • Local scope can be dividedd into two - Function scope and Block scope.

  • Function Scope
    • Whenever the a variable is declaraed inside a function, it is visible only inside the funtion.
    • var is the keyword used to define variable for a function-scope accessibility.

  • Block Scope
    • A block scope is the area between if and switch conditions or for and while loops.
    • Anything inside {curly brackets} is a block.
    • const and let are the keywords used to define variable in a block.

    function foo() {
      if(true) {
        var fruit1 = 'apple';           //exist in function scope
        const fruit2 = 'banana';       //exist in block scope
        let fruit3 = 'strawberry';     //exist in block scope
      }
      console.log(fruit1); //apple
      console.log(fruit2); //error: fruit2 is not defined
      console.log(fruit3); //error: fruit3 is not defined
    }

  • Lexical Scope
    • The children scope always has access to variable defined in the parent scope.
    • The children functions are lexically bound to the execution context of their parents.

Hoisting

  • Hoisting is Javascript's default behavior of moving all the declarations at the top.
  • Javascript hoists only declarations not intializations.

NOTE: Variable defined with let does not get hoisted.

Strict Mode

  • Strict mode is used on entire body or on a particular function.
  • It doesnot apply to any blocks with curly braces.
  • Type "use script"; in either global space or in function.
  • Javascript is a flexible language, where sometimes it does make assumptions. However, using strict mode, javascript throws error instead of assuming.
  • For example,
    • Declaration of variable; without strict mode if a variable is initialized without being declared then Javascript assumed declaration but in strict mode, an error is thrown.

this keyword

Rules to determine this

  • In a method, this refers to the owner object.
  • Alone, this refers to the global object.
  • In a function, this refers to the global object.
  • In a function, in strict mode, this is undefined.
  • In an event, this refers to the element that received the event.

Function call()

  • The call() method is used for an object that can use a method belonging to another object.

    •   var person = {
          fullName: function() {
                return this.firstName + " " + this.lastName;
            }
        }
        var person1 = {
            firstName:"John",
            lastName: "Doe"
        }
        person.fullName.call(person1); //returns "John Doe"

  • Parameters can also be added in the call function

    •   var person = {
          fullName: function(city, country) {
                return this.firstName + " " + this.lastName + ", " + city + ", " + country;
            }
        }
        var person1 = {
            firstName:"John",
            lastName: "Doe"
        }
        person.fullName.call(person1, "Oslo", "Norway"); //returns "John Doe, Oslo, Norway"

Function apply()

  • apply() method works similar to call() method. The only difference is apply() method taked paramters in form of array and call() method take them seperately.
    •   var person = {
          fullName: function(city, country) {
                return this.firstName + " " + this.lastName + ", " + city + ", " + country;
            }
        }
        var person1 = {
            firstName:"John",
            lastName: "Doe"

Functional Programming

Pure functions

Pure functions are the foundation of functional programming.

Pure functions has 2 characteristics:

  • Same inputs always give the same outputs.
  • Does No side-effects like
    • Mutating the input
    • console.log
    • HTTP calls (AJAX/fetch)
    • Changing of the file system
    • Querying the DOM

Pure functions does nothing rather than producing a return value by doing some calculations. It does not update another variable, mutate state, make http calls or query the DOM.

An application that uses more pure functions is easier to reason about, easier to unit test, can be memoized and can be parrallelised.

High order functions

.map()

With an example use .map() function.

var officers = [
  { id: 20, name: 'Captain Piett' },
  { id: 24, name: 'General Veers' },
  { id: 56, name: 'Admiral Ozzel' },
  { id: 88, name: 'Commander Jerjerrod' }
];

To get only the ids of the above array .map() function is most useful.

var officersIds = officers.map(function (officer) {
  return officer.id
});

Shortcut for the above code in Typescript and ES6:

const officersIds = officers.map(officer => officer.id);

Basically is takes 2 arguments, a callback and an optional context (will be considered as this in the callback). The callback runs for each value in the array and returns each new value in the resulting array.

.reduce() also runs a callback for each element of an array. What’s different here is that reduce passes the result of this callback (the accumulator) from one array element to the other.

The accumulator can be pretty much anything (integer, string, object, etc.) and must be instantiated or passed when calling .reduce().

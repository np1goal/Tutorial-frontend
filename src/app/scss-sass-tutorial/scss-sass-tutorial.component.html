Variables

SASS stores values of strings, numbers, colors, booleans, lists and nulls in Variables.

Variables can be created using $ before the variable.

For Example, $primary-color: #230978;

Used as, background-color: $primary-color;

Sass variables can be overriden.

Sass variables are only available at the level of nesting.

The variable can be oveeriden globally by using !global

Like, $myColor: green !global;

Nesting

Sass allows to nest CSS selectors

nav {
  ul {
    margin: 0;
    padding: 0;
    list-style: none;
  }
  li {
    display: inline-block;
  }
}

Sass can also have nested properties

font: {
  family: Helvetica, sans-serif;
  size: 18px;
  weight: bold;
}

& is a special character used in nesting to refer to parent selector.

@import

@import statement allows to import one file to another.

NOTE: CSS @import directive has a major drawback due to performance issues; it creates an extra HTTP request each time you call it. However, the Sass @import directive includes the file in the CSS; so no extra HTTP call is required at runtime.

@mixin

The @mixin directive lets you create CSS code that is to be reused throughout the website.

The @include directive is created to let you use the mixin.

@mixin name {
  property: value;
  property: value;
  ...
}

selector {
  @include mixin-name;
}

Passing Variables in Mixin

Mixin also accepts variables

@mixin bordered($color, $width) {
  border: $width solid $color;
}

.myArticle {
  @include bordered(blue, 1px); // Call mixin with two values
}

Default Values in Mixin

Mixin also accepts default values for variables

@mixin bordered($color: blue, $width: 1px) {
  border: $width solid $color;
}

.myTips {
  @include bordered($color: orange);
}

Mixin can also be used for Vendor Prefixes

Another good use of mixin is for vendor prefixes.

@mixin transform($property) {
  -webkit-transform: $property;
  -ms-transform: $property;
  transform: $property;
}

.myBox {
  @include transform(rotate(20deg));
}

@extend

The @extend directive is used to share a set of properties from one selector to another.

.button-basic {
  border: none;
  padding: 15px 30px;
  text-align: center;
  font-size: 16px;
  cursor: pointer;
}

.button-report {
  @extend .button-basic;
  background-color: red;
}

Map functions

An array of maps can also be created in SCSS. For Example,

$font-weights: ("regular": 400, "medium": 500, "bold": 700);

This is can be used as font-weight: map-get($font-weights, bold);

NOTE: SASS/SCSS Maps are immutable (cannot be changed).
Map function Explaination Example
map-get($map, $key) Gives value of the $key form the $list map-get(("red": #ff0000, "green": #00ff00), "green")
returns #00ff00
map-merge($map1, $map2) Merges two maps map-merge(("red": #ff0000, "green": #00ff00), ("blue": #0000ff))
returns ("red": #ff0000, "green": #00ff00, "blue": #0000ff)
map-remove($map, $keys) Returns map without given $keys map-remove(("red": #ff0000, "green": #00ff00), "red")
returns ("green": #00ff00)
map-keys($map) Returns all the keys of the map map-keys(("red": #ff0000, "green": #00ff00))
returns ("red", "green")
map-values($map) Returns all the values of the map map-values(("red": #ff0000, "green": #00ff00))
returns (#ff0000, #00ff00)
map-has-keys($map, $key) Returns boolean value after searching for the $key in the map map-keys(("red": #ff0000, "green": #00ff00), "blue")
returns false

Control Directives

@if statment

$a: 10;
@if($a > 10) {
  color: red;
} @else if ($a < 10) {
  color: green;
} @else }
  color: blue;
}

@while statment

$a: 10;
@while($a < 20) {
  .item-{#a} {
    color: green;
    $a = $a + 1;
  }
}

@for statment

through is >= or <=
to is = or <

@for $a from 1 though 20 {
  .item-{#a} {
    color: green;
    width: $a * 2px;
  }
}

@each statment

@each $s in (normal, bold, italic) {
  .#{$s} {font-weight: $s;}
}
